Reproduction kit for evaluating the robustness of **TSONet** under simulated spatial-resolution degradation of PhiSat-2 imagery.

The experiments compare the original resolution of **4.75 m/pixel** with simulated resolutions of **10 m/pixel**, **15 m/pixel**, and **30 m/pixel**. The 10 m and 30 m conditions reproduce the sensor-resolution experiment described in the original TSONet study, while 15 m is included as an intermediate extension.

This reproduction kit is based on the original [SpectorSong/TSONet](https://github.com/SpectorSong/TSONet) repository. This repository contains only the experiment-specific configurations, code patch, logs, and plots; the complete model implementation should be obtained from the original repository.

## Repository structure

```text
.
├── README.md
├── configs/
│   ├── resolution_degradation.patch
│   ├── train/
│   └── test/
├── dataset/
└── results/
    ├── plots/
    └── raw_logs/
```

### `configs/`

Contains the files required to reproduce the experimental configurations.

- `resolution_degradation.patch`: changes applied to the original TSONet code to support spatial degradation during data loading.
- `train/`: training scripts for the evaluated spatial resolutions.
- `test/`: evaluation scripts for the trained models.

### `dataset/`

Reserved for dataset instructions or local organization.

The PHDataset itself is **not included** in this repository and should not be committed to Git. Download it from the official source provided by the original TSONet/PHDataset authors.

### `results/`

Contains the artifacts used to audit and present the experiments.

- `raw_logs/`: original evaluation logs generated by `test.py`.
- `plots/`: plots generated from the consolidated results.

## Experimental conditions

| Condition | Source GSD | Target GSD |
|---|---:|---:|
| Original | 4.75 m/pixel | 4.75 m/pixel |
| Degraded 10 m | 4.75 m/pixel | 10 m/pixel |
| Degraded 15 m | 4.75 m/pixel | 15 m/pixel |
| Degraded 30 m | 4.75 m/pixel | 30 m/pixel |

The optical image is downsampled to the size corresponding to the target GSD and then resized back to the original patch dimensions before being passed to the network. The reference height map is not degraded.

For a `256 × 256` input patch, the approximate intermediate sizes are:

| Target GSD | Intermediate size |
|---|---:|
| 4.75 m | 256 × 256 |
| 10 m | 122 × 122 |
| 15 m | 81 × 81 |
| 30 m | 41 × 41 |

```text
Python
PyTorch

NumPy
```

Use PyTorch and CUDA versions compatible with your GPU and with the original TSONet implementation.

## Applying the patch

From the root of a clean TSONet checkout, verify that the patch can be applied:

```bash
git apply --check /path/to/TSONet-resolution-replication/configs/resolution_degradation.patch
```

Apply it:

```bash
git apply /path/to/TSONet-resolution-replication/configs/resolution_degradation.patch
```

The patch adds support for:

```text
--source_gsd
--target_gsd
```

## Dataset

Keep the official PHDataset train, validation, and test splits unchanged for every experimental condition.

Example dataset path:

```text
/path/to/PHDataset
```

The degradation is performed in memory by the data loader. Separate degraded copies of the dataset are not required.

## Training

Training scripts are stored in:

```text
configs/train/
```

List the available scripts:

```bash
find configs/train -maxdepth 1 -type f -print | sort
```

Run a configuration with:

```bash
bash configs/train/<training-script>.sh /path/to/PHDataset
```

The configurations should keep the architecture, data splits, optimizer, training duration, and remaining hyperparameters fixed. Only the experiment name and target GSD should change between resolution conditions.

## Evaluation

Evaluation scripts are stored in:

```text
configs/test/
```

List the available scripts:

```bash
find configs/test -maxdepth 1 -type f -print | sort
```

Run a configuration with:

```bash
bash configs/test/<test-script>.sh /path/to/PHDataset
```

## Reported results

The main values were extracted from the `Total Metrics` line of each evaluation log.

| Resolution | MRE | MAE | RMSE | R² | Precision | IoU | Recall | F1-score |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 4.75 m | 0.5098 | 5.1983 | 8.2814 | -0.0416 | 0.4339 | 0.3653 | 0.6979 | 0.5351 |
| 10 m | 0.5317 | 5.1384 | 7.9991 | 0.0282 | 0.4131 | 0.3503 | 0.6976 | 0.5189 |
| 15 m | 0.5731 | 5.6549 | 8.7134 | -0.1531 | 0.4248 | 0.3348 | 0.6125 | 0.5017 |
| 30 m | 0.5718 | 5.5308 | 8.5326 | -0.1058 | 0.3958 | 0.3226 | 0.6356 | 0.4878 |

Lower MAE, RMSE, and MRE values indicate lower height-estimation error. Higher Precision, Recall, IoU, F1-score, and R² values are preferable.

The spatial metrics are derived from the predicted height map using a 2 m threshold to distinguish building and non-building pixels.

## Main observations

- IoU and F1-score decrease as the simulated spatial resolution becomes coarser.
- The 10 m experiment produced slightly lower MAE and RMSE than the local 4.75 m baseline.
- The 15 m and 30 m conditions produced larger height errors than the original-resolution experiment.
- The non-monotonic behavior of the height metrics may reflect smoothing effects and training variability and should not be interpreted as evidence that lower resolution is generally beneficial.

## Reproducibility limitations

Each spatial-resolution condition was trained once. The reported values therefore do not capture variability caused by:

- random weight initialization;
- batch ordering;
- stochastic optimization;
- nondeterministic GPU operations;
- differences in software and CUDA versions.

For stronger statistical conclusions, repeat each condition with multiple random seeds and report the mean, standard deviation, and confidence interval.
